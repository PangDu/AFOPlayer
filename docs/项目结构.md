# AFOPlayer 项目结构说明

本文档描述 [AFOPlayer](https://github.com/PangDu/AFOPlayer) 仓库的目录组织、应用源码分层以及与 CocoaPods 集成方式，便于新成员与维护者快速定位代码。

**最后更新：** 2026-04

---

## 1. 仓库顶层

```
AFOPlayer/                          # Git 仓库根目录
├── .gitattributes
├── .gitignore
├── docs/
│   └── 项目结构.md                 # 本说明（当前文件）
└── AFOPlayer/                      # iOS 工程目录（Xcode 工程与 Pod 配置所在）
    ├── AFOPlayer.xcworkspace       # 含 Pods 时须用此工作区打开工程
    ├── AFOPlayer.xcodeproj         # 主 App 工程
    ├── Podfile                     # CocoaPods 依赖声明
    ├── Podfile.lock                # 锁定解析版本（提交后可重现构建）
    ├── Pods/                       # 由 `pod install` 生成（是否纳入 Git 以团队约定为准）
    └── AFOPlayer/                  # App Target 源码与资源（见下文）
```

---

## 2. App 目标源码树（`AFOPlayer/AFOPlayer/`）

相对路径自 `AFOPlayer/AFOPlayer/`（即含 `AFOPlayer.xcodeproj` 的目录下的内层 `AFOPlayer` 文件夹）：

```
AFOPlayer/
├── AFOAppDelegate.h
├── AFOAppDelegate.m                # 应用级委托、路由等与启动相关逻辑入口之一
├── controllers/                    # 视图控制器
│   ├── AppDelegate.h
│   ├── AppDelegate.m
│   ├── AFOOPMainController+AFOAppURLCache.m   # 在线播放 URL 缓存（NSUserDefaults，key 与历史 Pod 一致）
│   └── tabBarController/
│       ├── AFOAppTabBarController.{h,m}
│       ├── AFOAppTabBarController+AFOAutoRotate.{h,m}
│       └── …
├── models/                         # 数据与组装/协调模型
│   ├── AFOAddControllerModel.{h,m}
│   └── AFOAppCompositionRoot.{h,m} # 依赖装配、组合根
├── views/                          # 独立视图组件
│   └── AFOAppWindow.{h,m}
└── resources/                      # 资源与工程配置
    ├── AFOPlayer.pch               # 预编译头
    ├── Info.plist
    ├── main.m
    ├── Base.lproj/LaunchScreen.storyboard
    ├── Assets.xcassets/            # 图片与 App Icon
    ├── AFOMetalShaders.metal       # Metal 着色器
    └── AFOPlayer.bundle/           # 打包资源（文案、素材等）
```

### 2.1 分层职责（简要）

| 目录 | 职责 |
|------|------|
| `controllers/` | 界面流程、Tab、旋转适配、与播放/路由相关的 UI 入口 |
| `models/` | 业务数据、`CompositionRoot` 等非 UI 核心对象 |
| `views/` | 可复用窗口或自定义视图 |
| `resources/` | plist、故事板、资源目录、Metal、 pch、入口 `main.m` |

播放解码、路由、UI 组件库等具体实现依赖 **CocoaPods 中的 AFO* 系列 Pod**，不在上述应用目录内重复实现。

## 3. Xcode 工程文件

| 路径 | 说明 |
|------|------|
| `AFOPlayer.xcworkspace` | 主推荐打开方式：同时包含 App 工程与 `Pods` 工程 |
| `AFOPlayer.xcodeproj` | 仅 App；单独打开通常无法完整链接 Pod 静态库 |
| `xcshareddata/xcschemes` | 共享 Scheme（如 `AFOPlayer`） |

用户态数据 `xcuserdata/` 一般由 `.gitignore` 忽略，不提交。

## 4. CocoaPods 依赖（`Podfile` 直接声明）

当前主 Target `AFOPlayer` 在 `Podfile` 中声明的直接依赖包括：

- `AFODelegateExtension`
- `AFOFoundation`
- `AFOGitHub`
- `AFOLANUpload`
- `AFOOnlinePlay`
- `AFOWaterfall`
- `AFOPlaylist`
- `AFORouter`
- `AFOSQLite`
- `AFOSchedulerCore`
- `AFOUIKIT`
- `AFOViews`
- `AFOFFMpeg`

**平台：** iOS `13.0` 起（以 `Podfile` 中 `platform` 为准）。

完整传递依赖与精确版本以 **`Podfile.lock`** 为准。真机链路在 `post_install` 中会额外链接 FFmpeg / x264 / xvid / libyuv、部分 System Framework 等（详见 `Podfile` 内 `iphoneos_ldflags` / `simulator_ldflags`）。

---

## 5. 本地构建流程（摘要）

1. 安装 CocoaPods（若尚未安装）。
2. 在 `AFOPlayer/`（含 `Podfile` 的目录）执行：

   ```bash
   cd AFOPlayer
   export LANG=en_US.UTF-8   # 避免部分环境下 UTF-8 报错
   pod install
   ```

3. 使用 Xcode 打开 **`AFOPlayer.xcworkspace`**，选择 Scheme **AFOPlayer** 编译运行。

若私有 Spec 或本地 Path 依赖缺失，`pod install` 可能失败，需按团队提供的 Spec 源或 `Podfile` 中的 `pod …, :path` 配置补齐。
